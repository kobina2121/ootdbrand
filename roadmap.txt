## Level 4 - Payment & Webhooks (Critical Path)

### Paystack Checkout Flow
1. Customer submits checkout form.
2. Backend creates Order(status=Pending) + Paystack transaction initialization.
3. Return Paystack authorization URL to frontend.
4. Redirect customer to Paystack.
5. Paystack returns customer to success/failure page.
6. Backend verifies transaction via Paystack verify endpoint.
7. Update order status:
   - Success if verified and amount matches
   - Failed otherwise

### Webhook Handler (Required)
- Endpoint: /api/webhooks/paystack
- Validate Paystack signature (x-paystack-signature) using secret
- Handle events idempotently (skip already processed reference)
- Re-verify transaction server-to-server before marking Success
- Persist webhook payload/audit record
- Return fast 200 responses to prevent retries loop

### Security & Reliability
- Never trust client callback alone
- Verify amount/currency/reference match DB order
- Use raw body parsing where needed for signature validation
- Add retry-safe order update logic (atomic update by reference + status guard)

Complexity: Very High
